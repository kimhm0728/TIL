## Demand Paging(요구 페이징)
- 프로그램 실행 시작 시 프로그램 전체가 아닌 `초기에 필요한 것들만` 물리 메모리에 적재하는 전략
- 가상 메모리 시스템에서 주로 사용되며, 페이지로 관리된다.
- `페이저(pager)` : 프로세스 내 개별 페이지를 관리. 실행에 필요한 페이지이지들만 읽어와 시간과 메모리를 최소화한다. 
- `페이지 부재 트랩(Page fault trap)` : 메모리에 올라와 있지 않은 페이지를 접근할 때 발생
 
</br> 
 
## 페이지 교체
1. 프로세스 동작에 필요한 페이지를 요청한다.
2. 물리 메모리에 해당 페이지가 있는지 확인한다.
3. 페이지가 없다면(페이지 부재) 원하는 페이지를 보조 저장장치에서 가져온다.
4. 물리 메모리가 모두 사용 중이라면, `페이지를 교체`한다. 

### 페이지 교체 흐름
1. 디스크에서 필요한 페이지의 위치를 찾는다
2. 빈 페이지 프레임을 찾는다.
  1. `페이지 교체 알고리즘`을 통해 희생될(victim) 페이지를 고른다.
  2. 희생될 페이지를 디스크에 기록하고, 관련 페이지 테이블을 수정한다.
3. 새롭게 비워진 페이지 프레임에 새 페이지를 읽어오고, 프레임 테이블을 수정한다.
4. 사용자 프로세스 재시작
 
</br> 
 
## 페이지 교체 알고리즘
### 1. FIFO(First-In-First-Out)
- 가장 간단한 페이지 교체 알고리즘으로, `물리 메모리에 들어온 페이지 순서대로` 교체된다.
- 장점
  - 이해하기 쉽고, 프로그램하기 쉽다.
- 단점
  - 오래된 페이지가 항상 불필요하지 않은 정보를 포함하는 것은 아니다. (초기 변수)
  - 활발하게 사용되는 페이지를 교체해 페이지 부재율을 높일 수 있다.
  - `Belady의 모순` : 페이지를 저장할 수 있는 페이지 프레임의 수를 늘려도 페이지 부재가 더 많이 발생
### 2. OPT(Optimal Page Replacement, 최적 페이지 교체)
- `앞으로 가장 오랫동안 사용되지 않을 페이지`를 교체한다.
- 장점
  - Belady의 모순이 발생하지 않는다.
  - 알고리즘 중 페이지 부재율이 가장 낮다.
- 단점
  - 모든 프로세스의 메모리 참조 계획을 파악할 수 없어 구현이 어렵다.
### 3. LRU(Least Recently Used)
- `가장 오랫동안 사용되지 않은 페이지`를 교체한다.
- `FIFO`보다 성능이 우수하고, `OPT`보다는 성능이 떨어진다.
### 4. LFU(Least Frequently Used)
- `참조 횟수가 가장 적은 페이지`를 교체한다.
- 어떤 프로세스가 특정 페이지를 집중적으로 사용하다 다른 기능을 사용하게 되면, 더이상 사용하지 않아도 계속 메모리에 머물게 된다.
- OPT를 제대로 근사하지 못하여 잘 쓰이지 않는다.
### 5. MFU(Most Frequently Used)
- `참조 횟수가 가장 많은 페이지`를 교체한다.
- 참조 횟수가 가장 적은 페이지가 최근에 메모리에 올라왔고, 앞으로 계속 사용될 것이라는 가정에 기반한다.
- OPT를 제대로 근사하지 못하여 잘 쓰이지 않는다.
